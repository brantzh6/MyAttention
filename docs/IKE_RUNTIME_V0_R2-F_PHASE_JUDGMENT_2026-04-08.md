# IKE Runtime v0 R2-F Phase Judgment

## Phase

`R2-F = Visible Benchmark Queue Bridge Narrow Integration`

## Why This Phase Exists

After `R2-E`, runtime bootstrap and settings-surface activation are both real.

The next narrow gap is not broader UI/runtime integration.
It is one explicit action path from the current visible benchmark story into runtime review:

- reviewed benchmark candidate
- explicit user action
- runtime `pending_review` packet
- no auto-promotion to trusted memory

## Guardrails

- require an existing runtime project
- keep runtime bootstrap explicit
- keep benchmark import explicit
- import only reviewed candidate payloads
- do not auto-accept packets
- do not merge knowledge-base truth with runtime truth

## Intended Proof

One controller-facing settings surface can now:
- read benchmark candidate state
- explicitly send that candidate into runtime review
- preserve the runtime trust gate
