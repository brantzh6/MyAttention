## 2. 核心判断

这个项目最有价值的，不是“写更长的研究报告”。

最有价值的是：

- `research-assets-first`
- `source-plan-first`
- `fact / interpretation separation`
- `boundary / tension review`
- `structured handoff`

也就是说，它真正提供的是一种**深度研究骨架**，而不是一份搜索总结模板。

这对 MyAttention 有价值，而且应该进入主线设计。

## 3. 它真正做对了什么

### 3.1 固定骨架，变化模式

它的方法不是每次自由发挥，而是先走固定骨架：

1. task framing
2. mode selection
3. source planning
4. evidence collection
5. mode-specific analysis
6. boundary review
7. implication synthesis
8. structured handoff

这点很强，因为它同时避免了两种常见失败：

- 研究漂移
- 单模板套所有任务

### 3.2 先研究资产，后报告

它把报告降成下游产物，而不是唯一结果。

这一点特别适合 MyAttention，因为我们要的不是一次性文章，而是：

- 可复用知识
- 可复用决策支持
- 可复用方法
- 可沉淀长期记忆的结构化资产

### 3.3 source plan 是研究前置步骤

这与我们最近的判断是高度一致的：

- 来源发现和来源执行必须分开
- 不能一上来就 opportunistic browsing
- 不能把 source quality 当成默认成立

### 3.4 boundary / tension 是一等公民

这套方法要求明确写出：

- trade-offs
- counterarguments
- non-fit scenarios
- uncertainty

这一点很重要，因为它和 MyAttention 的三个核心能力都直接相关：

- 决策支持不能只有单向结论
- 知识系统不能只保留“看起来对”的叙述
- 进化大脑不能只看“通过/失败”，还要看边界和误判风险

### 3.5 handoff 不是 prose，而是对象

它要求下游接收：

- core claims
- key evidence
- tensions
- boundaries
- chapter / page candidates
- open questions

这意味着研究结果不是写完就结束，而是能继续进入：

- storyline
- memo
- strategy
- decision support

这一点对 MyAttention 的 dialog brain、chief brain、knowledge brain 都很有参考价值。

## 4. 哪些地方不能直接照搬

### 4.1 不能把 MyAttention 变成 PPT 上游工具

被参考的项目主要面向“深度研究 -> 下游叙事/汇报”。

MyAttention 比它大得多，还包括：

- 持续信息获取
- source intelligence
- 世界知识体系建设
- 长期记忆
- 自我进化

所以我们应该吸收它的方法纪律，而不是复制它的产品边界。

### 4.2 投票不能替代深度研究

我直接纠偏这一点：

“深度研究方法 + 投票”是一个可能的组合，
但**投票不能成为寻找最佳答案的主骨架**。

更合理的关系是：

- 深度研究骨架在前
- 投票是其中的分析、比较、复核、分歧提炼工具

投票适合用在：

- comparative analysis
- decision research
- boundary review
- evolution review

但它不能替代：

- source plan
- evidence log
- fact base
- interpretation map
- boundary review

### 4.3 MyAttention 不是一次性 research bundle 系统

被参考方法适合离散研究任务。

MyAttention 还必须覆盖：

- 连续任务
- 持续采集
- 热/温/冷知识对象
- 订阅源的持续更新
- 方法与 skill 的持续进化

所以后面一定要把“per-task deep research”放进更大的持续循环中。

## 5. 对 MyAttention 的直接启发

## 5.1 可以吸收的深度研究主骨架

对于高价值研究任务，MyAttention 应使用：

1. `research_brief`
2. `research_mode_selection`
3. `source_plan`
4. `evidence_log`
5. `fact_base`
6. `interpretation_map`
7. `boundary_map`
8. `tension_map`
9. `implication_map`
10. `decision_or_story_handoff`

这套结构非常适合进入我们的主线设计。

## 5.2 投票在什么位置最有价值

投票不应在第一步出现。

它最适合在这些环节出现：

- 研究模式中的 `comparative_analysis`
- 研究模式中的 `decision_research`
- `boundary review`
- `evolution brain` 的 root cause / recovery option review

也就是说：
**投票不是取代研究，而是帮助研究做多视角分析与复核。**

## 5.3 这套方法为什么比“搜索 + 总结”更适合我们

因为它强制我们回答：

- 研究任务到底是什么
- 这个研究服务谁
- 哪些来源最重要
- 哪些是事实，哪些只是解释
- 哪些地方存在张力和边界
- 最后如何把结果结构化交给下游

这正好能解决我们最近一直暴露出的几个问题：

- 方向不够清楚
- 研究过程不稳定
- 研究资产不沉淀
- 投票结果容易变成套话总结

## 6. 对当前主线的影响

这份方法研究，应该直接影响当前主线的后续设计。

### 6.1 对 source intelligence 的影响

source intelligence 不能只解决“从哪里抓”。
它还要服务于 `source_plan`。

### 6.2 对 memory 的影响

研究结果不应该只变成聊天答案。
它应该沉淀成：

- evidence memory
- fact memory
- interpretation memory
- procedural memory

### 6.3 对 evolution brain 的影响

进化大脑也不应该只是监控和告警。
它后面应该复用同样的方法纪律：

- evidence first
- interpretation second
- boundary before action

## 7. 当前结论

应当把这套深度研究方法作为 MyAttention 主线设计的重要参考。

但正确的吸收方式不是：

- “以后都用投票来做研究”

而是：

- 用深度研究骨架作为方法底座
- 用投票作为其中的比较、复核、分歧提炼工具
- 再把这套方法嵌入持续采集、长期记忆和多大脑协作之中

## 8. 建议下一步

在进入主线设计前，应该把这份研究结果继续收敛进：

- `docs/SOURCE_INTELLIGENCE_ARCHITECTURE.md`

这样后面我们进入主线，就不是“继续搜一搜再总结”，而是有明确研究方法底座了。
