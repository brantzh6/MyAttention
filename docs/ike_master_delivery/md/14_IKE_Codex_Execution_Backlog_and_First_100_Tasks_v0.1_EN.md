# IKE Codex Execution Backlog and First 100 Tasks v0.1

## 1. Purpose

This document translates the architecture into an execution backlog suitable for Codex-driven implementation.

## 2. Task Buckets

The first 100 tasks are grouped into: repository scaffold, shared schemas, persistence, observation ingestion, knowledge assembly, model governance skeleton, workflows, task/experiment/decision engine, harness, API surface, and ops.

## 3. Sequencing Rule

Complete the narrowest end-to-end slice first: observation → entity/claim extraction → research task → experiment stub → decision record → harness record.

## 4. Delivery Rhythm

Codex should work in small batches with contract-first commits and mandatory harness updates when interfaces change.
