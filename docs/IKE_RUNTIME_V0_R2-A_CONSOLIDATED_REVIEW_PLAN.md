Turn the current runtime-v0 hardening stack into one truthful consolidated
readiness package.

## Scope

- phase/result synthesis
- controller judgment hardening
- concise review-pack preparation
- closure-discipline settlement for the most important carried review debt

## Required Output

1. one consolidated runtime-v0 review milestone
2. one concise controller/cross-model review prompt or execution pack
3. one explicit list of:
   - materially complete areas
   - mixed-evidence areas
   - remaining blockers
4. one explicit debt-settlement outcome for:
   - `force=True`
   - retained notes
   - `R1-J` method rule
5. one explicit scheduling decision for:
   - second concept benchmark
   - procedural memory evolution

`R2-A` is acceptable only if:

- it does not invent new runtime semantics
- it clearly distinguishes completed work from watch items
- it closes or explicitly reclassifies the major carried review debt
- it gives a truthful next-step basis for either:
  - a narrow broader integration gate
  - or another narrow hardening phase, if actually justified
