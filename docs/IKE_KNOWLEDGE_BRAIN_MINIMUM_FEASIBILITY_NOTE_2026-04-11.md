# IKE Knowledge Brain Minimum Feasibility Note

Date: 2026-04-11
Status: controller feasibility note

## Purpose

Define the minimum truthful launch condition for a first narrow `Knowledge
Brain` packet.

This document is not a full knowledge architecture.

It exists to prevent the project from waiting for every upstream system to be
"finished" before any bounded Knowledge Brain work can begin.

## Core Judgment

The project does not need:

- perfect Source Intelligence
- perfect Runtime v0 closure
- complete memory layering
- full ontology/graph design

before starting the first narrow `Knowledge Brain` feasibility slice.

But it does need enough truthful substrate that the first packet is not fake.

## Minimum Launch Condition

A first Knowledge Brain feasibility packet may start when all of the following
are true:

1. there is at least one durable feed/data substrate worth querying
2. there is at least one bounded retrieval/read surface
3. the packet is explicit that it is feasibility-only, not canonical knowledge
   truth
4. the output stays narrow:
   - candidate knowledge synthesis
   - candidate fact grouping
   - candidate entity/topic structuring
   not broad autonomous reasoning claims

## Current Project Readiness

Current project assets already make a narrow first packet plausible:

- feed/data substrate exists
- runtime truth kernel is materially real
- memory retention rules are now explicit
- knowledge and memory research documents already exist

This means the blocker is no longer "nothing exists."

The blocker is choosing one bounded first packet.

## Recommended First Packet Shape

The first Knowledge Brain feasibility slice should likely be:

- one bounded topic or domain
- one bounded source set or feed slice
- one explicit output such as:
  - entity grouping
  - fact clustering
  - topic timeline
  - reviewed candidate knowledge note

## Non-Goals

- no grand knowledge graph rollout
- no claim of universal semantic truth
- no attempt to solve all memory, ontology, and reasoning at once
- no bypass of runtime/controller review

## Relation To Runtime And Source Intelligence

Runtime v0 remains the operational kernel.

Source Intelligence V1 remains the next product-capability start line for
upstream source quality.

Knowledge Brain should start only as a narrow feasibility packet that consumes
existing substrate honestly.

## Recommendation

`accept`

The project should later open one bounded Knowledge Brain feasibility packet,
but it should remain below Runtime v0 and Source Intelligence V1 in current
priority.
