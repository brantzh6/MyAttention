Date: 2026-04-11
Status: controller phase judgment

## Purpose

Turn `Source Intelligence` from a known strategic bottleneck into a bounded
implementation-start line.

This document does not re-argue the whole architecture.
It fixes the phase boundary, immediate scope, and start condition.

## Current Controller Judgment

`Source Intelligence V1` should now become the next product-capability start
line.

This does not replace `IKE Runtime v0` as the true active mainline.

It means:

- `Runtime v0` remains the kernel/mainline
- `Source Intelligence V1` becomes the highest-priority capability-start line
- support tracks should serve these two realities instead of opening more
  parallel fronts

## Phase Objective

Deliver one narrow `V1/M1` capability:

- given a topic or research target
- produce a candidate source/object set
- classify each candidate at a bounded V1 level
- recommend monitoring/review/execution strategy
- persist or at least structure the result in a way that can later connect to
  source-plan truth

## In Scope For V1/M1

- topic-driven source discovery entry
- candidate object / candidate source result shape
- bounded V1 classification fields
- strategy recommendation fields
- one narrow execution path or service boundary
- one reviewable output packet

## Explicitly Out Of Scope

- rewriting the full fetcher stack
- replacing `feed_sources.json` globally in one pass
- broad UI redesign
- complete memory-system integration
- full source evolution automation
- claiming research-grade source quality immediately

## Phase Entry Condition

This phase can start now because:

- research exists
- architecture exists
- strategic review absorption has already accepted the need for an
  implementation start packet

No additional concept-only phase is required first.

## Phase Exit Condition

This start phase is complete when the project has:

1. one bounded implementation-start packet
2. one explicit M1 scope definition
3. one coding brief for delegated implementation
4. one review pack for external or delegated review
5. one clear next coding lane that does not require reconstructing the whole
   history from chat

## Recommendation

`accept`

The project should proceed directly to a bounded `Source Intelligence V1`
implementation-start packet.
