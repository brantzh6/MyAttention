## Purpose

This pack is for a broad external review of the current project strategy after
more than one month of work.

The review should focus on whether the project has expanded too far, what
should be compressed, what should be externally adapted rather than self-built,
and whether the current mainline still makes sense.

## Review Questions

Please review the current project through these questions:

1. Has IKE expanded into too many parallel hard problems at once?
2. Which lines are core and should remain self-owned?
3. Which lines should be adapted from external systems rather than built from
   scratch?
4. Which lines should be compressed, deferred, or de-scoped?
5. Is the current active mainline still correct?
6. Does the project currently risk "scientific perfectionism without closure"?
7. What concrete priority order would you recommend for the next 2 to 4 weeks?

- controller/governance shell
- source intelligence preparation
- agent harness hardening

The concern is that the project may be architecturally correct in many places
but too broad to close.

These are the current proposed principles for review:

1. IKE should own judgment, truth, governance, and absorptive intelligence.
2. IKE should not rebuild every execution substrate itself.
3. External agent systems may be used as execution/workbench layers, but not
   as IKE's final controller or canonical truth.
4. Runtime remains the true mainline.
5. Source Intelligence is the next product-capability start line.
6. Support and research tracks must not silently become co-equal mainlines.
7. Large ambitions should remain north-star directions unless reduced to
   bounded packets.
