# IKE B2 Concept Trigger Plan

## Purpose

This document defines the next benchmark stage after the current B1 harness
benchmark.

B1 has already proven:

- signal detection
- meaningful entities
- an initial meaning summary
- an initial relevance judgment

The next step is not more ranking polish.
It is a stronger concept-and-action layer.

## B2 Goal

B2 should turn a concept from:

- `interesting and relevant`

into:

- `better defined`
- `better contrasted`
- `mapped to explicit project gaps`
- `capable of triggering one bounded research/prototype action`

## B2 Required Outputs

For a concept such as `harness`, B2 should eventually produce:

1. `concept card`
   - concise concept definition in current context
   - main interpretations or variants
   - confidence and limitations

2. `entity tiers`
   - `concept-defining`
   - `ecosystem-relevant`
   - `implementation-relevant`

3. `concept boundary comparison`
   - `harness vs benchmark`
   - `harness vs unit test`
   - `harness vs skill`
   - `harness vs guardrail`

4. `mainline gap mapping`
   - which current project gap(s) it can help close

5. `research recommendation level`
   - `observe`
   - `study`
   - `prototype`
   - `adopt_candidate`

6. `one bounded research/prototype trigger`
   - not a strategy rewrite
   - not a vague watchlist action

## Why B2 Exists

Without B2, benchmark outputs remain:

- better than search
- but weaker than a true project-moving intelligence loop

B2 is where the benchmark starts to become:

- a research trigger
- a method adoption filter
- a real evolution-brain precursor

## Guardrails

B2 should not:

- become a generic trend platform
- become a broad ontology platform
- assume the system already has perfect world knowledge
- skip evidence grounding
- jump directly to large implementation plans

## Immediate Use

B2 now becomes the controller target after B1.

Future B2 packets should prioritize:

1. concept-deepening artifact shape
2. entity tiering
3. gap mapping
4. bounded trigger recommendation
