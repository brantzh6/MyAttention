## Goal

Close the last narrow gate items before any broader runtime integration is
considered.

## Scope

`R2-B` stays inside:

- narrow lifecycle proof
- narrow kernel-to-benchmark proof
- remaining gate-accounting work
- explicit long-horizon scheduling

## Subtracks

### R2-B1 Debt Recovery Follow-Through

Focus:

- make sure the carried gate items are no longer implicit
- prepare the runtime side for a truthful next proof

### R2-B2 First Real Task Lifecycle Proof

Focus:

- task reaches done through the current task/claim/review/decision machinery

- feed one benchmark-originated structured output into the runtime kernel
- prove the kernel can consume it without widening the platform

Success standard:

- one benchmark-originated object/result becomes a truthful runtime input
- the bridge remains narrow, explicit, and auditable

### R2-B4 Formal Scheduling Of Long-Horizon Items

Focus:

- second concept benchmark

Success standard:

- both receive formal planned placement
- neither remains an implicit carried note

## Acceptance Standard

`R2-B` is acceptable only if:

- it does not widen platform scope
- it produces one truthful lifecycle proof

- it converts the repeatedly deferred long-horizon items into explicit schedule
