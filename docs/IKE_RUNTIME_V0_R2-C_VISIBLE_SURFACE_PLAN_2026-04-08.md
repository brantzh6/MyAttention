# IKE Runtime v0 R2-C Visible Surface Plan

Date: 2026-04-08
Phase: `R2-C`
Title: `Runtime-to-Visible-Surface Narrow Integration`

## Objective

Prove one narrow runtime-backed visible surface without widening into broad
UI/runtime integration.

## Scope

- one runtime-backed project/work read helper
- one narrow API route
- one settings-facing runtime panel
- truthful unavailable state when runtime project truth is absent

## Non-Goals

- no broad UI integration
- no knowledge-base/runtime unification
- no benchmark auto-promotion to trusted memory
- no notification or graph-memory work

## Closure Standard

`R2-C` is complete when:

1. one runtime-backed visible surface is real and tested
2. review/testing/evolution are durably recorded
3. controller judgment remains narrow and non-expansive
