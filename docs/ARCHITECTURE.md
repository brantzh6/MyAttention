### 存储层扩展说明

当前主架构中的存储层不仅包括 PostgreSQL、Qdrant、Redis，还预留了对象存储能力：

- 开发阶段：本地文件系统，通过 `LocalObjectStore` 实现
- 生产/扩展阶段：MinIO / S3 / OSS，通过统一对象存储接口实现

对象存储用于承载：

- 原始 RSS / HTML / JSON
- PDF / 图片 / 截图等大对象
- 中间处理产物与快照

也就是说，长期目标中的存储层应理解为：

- PostgreSQL：结构化事实与聚合分析
- Qdrant：语义向量与相似性检索
- Redis：缓存、队列缓冲、调度状态
- Object Storage：原始对象与大文件
