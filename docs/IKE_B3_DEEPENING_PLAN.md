# IKE B3 Deepening Plan

## Purpose

B2 has already proven that a real-world concept can be turned into:

- entity tiers
- explicit mainline gap mappings
- a recommendation level
- a bounded research trigger packet

The next stage is not more ranking polish.

It is:

- concept deepening
- mechanism-to-gap explanation
- applicability judgment

## Why B3 Exists

B2 answers:

- what concept matters
- which entities matter
- what next bounded research task should happen

But B2 still does not answer strongly enough:

- what the concept *really is* in this context
- what it is *not*
- which competing interpretations exist
- how it improves a specific project gap
- which IKE layer it best applies to

That is the purpose of B3.

## B3 Goal

Turn a concept from:

- `research trigger candidate`

into:

- `better defined`
- `boundary-aware`
- `mechanism-mapped`
- `applicability-judged`

## Required B3 Outputs

For a concept such as `harness`, B3 should produce:

1. `working definition`
   - the current best-fit definition for this concept in the current context

2. `boundary / non-boundary`
   - what the concept includes
   - what it does not include

3. `competing interpretations`
   - plausible alternative meanings in the same attention neighborhood

4. `current best-fit interpretation`
   - which interpretation is most supported now
   - why

5. `mechanism-to-gap mapping`
   - not just `which gap it touches`
   - but `how it could improve that gap`

6. `applicability judgment`
   - which IKE layer it is most relevant to:
     - information layer
     - knowledge layer
     - evolution layer
     - active work surface
     - task / experiment loop
     - evaluation infrastructure

## Guardrails

B3 should not:

- become a broad ontology platform
- become a generic method encyclopedia
- assume the concept is already proven adoptable
- claim production value before the study task is closed

## Immediate Use

B3 is the next conceptual layer after B2.

The expected sequence is:

- B1: signal + meaning + relevance hint
- B2: concept trigger + study packet
- B3: concept deepening + applicability review

## Relation To Task Closure

B3 should inform, but not replace, the task closure path.

Task closure answers:

- what the study found
- whether to prototype / defer / reject

B3 answers:

- what the concept means deeply enough to justify that decision
