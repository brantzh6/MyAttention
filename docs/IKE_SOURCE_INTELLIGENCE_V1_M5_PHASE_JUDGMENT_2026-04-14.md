# IKE Source Intelligence V1 - M5 Phase Judgment

Date: 2026-04-14
Phase: `M5 Contextual Media Article Signal Classification`
Status: `approved_for_bounded_start`

## Purpose

Open one bounded post-`M4` quality slice that improves object quality rather
than overlap compression.

## Why M5 Starts Now

The current discovery path can already:

- preserve continuity
- form one bounded loop
- compress two obvious overlap families

The next useful narrow step is to improve how one valuable real-world source
family is classified.

## M5 Objective

For contextual tech-media article pages such as `36kr` article URLs:

1. classify them as `signal`
2. only in `LATEST` / `FRONTIER`
3. keep `METHOD` unchanged

## Explicit Non-Goals

- no broad media ranking redesign
- no generic article parser
- no publisher-wide trust-model rewrite
- no source-plan schema changes

## Exit Condition

`M5` is complete when the project has:

1. one bounded article classification rule
2. one helper proof and one route proof
3. one explicit truth boundary that this is a contextual-media quality slice,
   not a general media-source ontology
