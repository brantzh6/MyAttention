# IKE Meta-Reasoning and Model Governance Spec v0.1

## 1. Scope

This specification defines how thinking models are registered, selected, evaluated, downgraded, frozen, retired, or newly admitted.

## 2. Core Objects

Core governance objects include ThinkingModel, DomainParadigm, ModelSelectionRecord, ModelEvaluationRecord, GovernanceReview, and ModelEvolutionEvent.

## 3. Selection Policy

Selection should consider task type, field, evidence shape, risk level, confidence state, and historical performance. Selection should be explainable and logged.

## 4. Update Policy

Model updates happen at four levels: parameter-level tuning, combination-level updates, model admission/removal, and meta-policy changes.

## 5. Governance Cadence

Run weekly operational reviews, monthly model reviews, and quarterly architecture reviews. Governance should be routine, not ad hoc.
