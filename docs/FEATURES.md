# MyAttention 功能文档

## 概述

MyAttention 是一个 AI 驱动的智能决策支持系统，旨在帮助用户获取、整理和分析信息，支持重大决策。

---

## 核心功能模块

### 1. 智能对话系统

**路径**: `/chat`

#### 1.1 多模型支持

| 模型 | 提供商 | 特点 | 支持功能 |
|------|--------|------|----------|
| Qwen Max | 阿里云 | 中文能力强 | 联网搜索 |
| Qwen 3.5 Plus | 阿里云 | 均衡性能 | 联网搜索 + 深度思考 |
| DeepSeek V3.2 | DeepSeek | 性价比高 | 联网搜索 + 深度思考 |
| GLM-5 | 智谱AI | 国产优秀 | 深度思考 |
| MiniMax M2.5 | MiniMax | 长文本 | 联网搜索 |
| Kimi K2.5 | 月之暗面 | 长上下文 | - |

#### 1.2 对话模式

- **普通模式**: 单模型对话，自动选择或手动指定模型
- **投票模式**: 多模型并行回答，综合得出共识结论
- **联网搜索**: 支持实时网络信息检索（部分模型）
- **深度思考**: 启用模型推理过程展示（部分模型）

#### 1.3 知识库检索 (RAG)

- **开关控制**: 可开启/关闭知识库检索
- **多知识库选择**: 可选择搜索特定知识库或全部知识库
- **来源展示**: 回答中展示引用的知识库来源

#### 1.4 对话历史

- **会话管理**: 自动创建和保存对话
- **历史记录**: 侧边栏显示所有历史对话
- **上下文保持**: 自动保持对话上下文

---

### 2. 知识库管理系统

**路径**: `/settings/knowledge`

#### 2.1 知识库类型与区别

| 知识库类型 | 用途 | 适用场景 |
|------------|------|----------|
| **通用知识库** | 存储各类文档 | 个人笔记、参考资料 |
| **专业领域库** | 特定领域文档 | 技术文档、法律条款 |
| **项目知识库** | 项目相关资料 | 项目文档、会议记录 |
| **订阅内容库** | RSS/Feed内容 | 新闻、博客文章 |

#### 2.2 知识库操作

**创建知识库**
- 名称和描述
- 自动分配唯一 ID
- 创建 Qdrant 向量集合

**导入内容**
- **文件上传**: 支持 TXT、PDF、DOCX、MD、HTML
- **网页索引**: 输入 URL 自动抓取
- **网络搜索**: 阿里云 IQS 搜索并自动索引

**管理文档**
- 查看文档列表和预览
- 删除单个文档
- 查看知识库统计（文档数、分块数）

#### 2.3 知识库在对话中的使用

1. **聊天界面**: 点击"知识库"按钮旁的下拉菜单
2. **选择范围**:
   - "全部知识库": 搜索所有知识库
   - 选择特定知识库: 仅搜索选中的知识库
3. **发送问题**: 系统自动检索相关内容并作为上下文

#### 2.4 文档处理流程

```
上传文件 → 解析内容 → 分块处理 → 向量化 → 存储到 Qdrant
         ↓
   支持格式: TXT, PDF, DOCX, MD, HTML
         ↓
   分块策略: 500字符/块, 50字符重叠
         ↓
   向量模型: all-MiniLM-L6-v2 (384维)
```

### 3. 信息流系统

**路径**: `/` (首页)

#### 3.1 信息源管理

**支持的信息源类型**
- **RSS/Atom**: 标准 RSS 订阅
- **Web 监控**: 网页内容监控
- **API 源**: GitHub、HackerNews 等

**信息源配置**
- 名称、URL、分类
- 优先级设置（高/中/低）
- 启用/禁用开关
- 刷新频率

#### 3.2 信息流展示

- **时间线视图**: 按时间排序
- **来源分类**: 按信息源分组
- **重要性排序**: AI 评估重要性
- **快速操作**: 标记已读、收藏、删除

### 4. 记忆系统

**路径**: `/settings/memory`

#### 4.1 记忆类型

| 类型 | 说明 | 示例 |
|------|------|------|
| **偏好 (Preference)** | 用户偏好设置 | "用户喜欢简洁的回答" |
| **事实 (Fact)** | 重要事实记录 | "用户是软件工程师" |
| **决策 (Decision)** | 决策记录 | "用户决定使用 React 框架" |
| **洞察 (Insight)** | 总结和洞察 | "用户关注 AI 领域" |

#### 4.2 记忆功能

- **自动提取**: 从对话中自动提取重要信息
- **手动添加**: 用户主动添加记忆
- **记忆检索**: 对话时自动检索相关记忆
- **记忆管理**: 查看、编辑、删除记忆

## API 接口概览

### 对话接口

```
POST /api/chat
Body: {
  message: string,
  conversation_id?: string,
  use_voting: boolean,
  use_rag: boolean,
  enable_search: boolean,
  enable_thinking: boolean,
  provider?: string,
  model?: string,
  kb_ids?: string[]
}
```

### 知识库接口

```
GET  /api/rag/knowledge-bases          # 列出所有知识库
POST /api/rag/knowledge-bases          # 创建知识库
DELETE /api/rag/knowledge-bases/{id}   # 删除知识库
POST /api/rag/knowledge-bases/{id}/documents/file  # 上传文件
POST /api/rag/knowledge-bases/{id}/documents/web   # 索引网页
GET  /api/rag/knowledge-bases/{id}/search          # 搜索
```

### 对话历史接口

```
GET  /api/conversations               # 获取对话列表
GET  /api/conversations/{id}/messages # 获取对话消息
DELETE /api/conversations/{id}        # 删除对话
```

### 记忆接口

```
GET  /api/memories/profile   # 获取用户画像
GET  /api/memories           # 获取记忆列表
POST /api/memories           # 添加记忆
DELETE /api/memories/{id}    # 删除记忆
```
