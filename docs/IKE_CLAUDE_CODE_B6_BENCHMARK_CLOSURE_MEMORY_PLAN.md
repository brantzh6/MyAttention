# IKE Claude Code B6 Benchmark Closure Memory Plan

## Purpose

`B6` defines the first truthful upstream producer of procedural-memory payloads:

- reviewed benchmark study closure

This follows the accepted conclusion that procedural-memory v1 should not be
fed by heuristic generic task closure automation.

## Main Decision

First payload source:

- reviewed benchmark `study_closure`

Not first:

- generic task closure automation
- generic decision handoff

## Why This Source Wins

- bounded and inspectable
- already review-shaped
- naturally contains findings and lessons
- low risk of synthetic memory spam

## Required Fields

The benchmark closure flow should explicitly provide:

- `title`
- `lesson`
- `why_it_mattered`
- `how_to_apply`
- `confidence`
- `source_artifact_ref`

These fields should be authored or confirmed at review/closure time.

## Success Condition

`B6` succeeds if IKE can produce explicit procedural-memory payloads from
reviewed benchmark closures without heuristic content invention.
