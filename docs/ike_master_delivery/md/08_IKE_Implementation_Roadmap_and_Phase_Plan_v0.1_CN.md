# IKE 实施路线图与阶段计划 v0.1

## 1. 路线原则

明确采用 A→D 的演化路径。A/B 是现实起点，D 是长期目标，必须写下来，避免后续开发只做局部最优。

## 2. 阶段规划

M0：原则与启动准备；M1：对象模型 + 工作流 + 初始 harness；M2：模型治理与范式包；M3：实验与决策闭环；M4：portfolio 式模型治理。

## 3. 三条优先建设流

优先建设流包括：对象/Schema 流、工作流/任务引擎流、Harness/治理流。

## 4. 避免偏航

不要让系统退化成 crawler + vector DB + chat wrapper。长期目标必须始终保持为“自我改进的研究系统”。
