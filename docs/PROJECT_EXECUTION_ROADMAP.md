# 项目推进路线图

## 目标

把已有部分能力的 MyAttention，从“局部可用”推进到“围绕最终目标持续演进的完整系统”。

最终目标仍然分成三条主线：

- 信息大脑
- 知识大脑
- 进化大脑

## 当前阶段判断

当前不是重新设计方向，而是把已经确定的架构方案逐步接入现有代码。

当前最紧急的两条线是：

1. 让信息大脑的事实层和存储层真正稳定。
2. 让进化大脑从辅助能力变成正式工作台和质量闭环入口。

## 阶段路线

### Phase 1：对象存储与原始层

目标：

- 建立原始对象层和可扩展存储抽象。

状态：

- 已完成。

已落地：

- `ObjectStore`
- `LocalObjectStore`
- `raw_ingest`
- `/api/feeds/import` 原始层持久化

### Phase 2：信息流事实层落库

目标：

- 让 `feed_items` 成为可持续使用的事实层，而不是只依赖运行时缓存。

状态：

- 部分完成。

已落地：

- `/api/feeds/import` 写入 `raw_ingest + sources + feed_items + cache`
- `/api/feeds` 支持 `cache / db / hybrid`

未完成：

- 事实层完全切主
- 围绕事实层的质量指标和富化闭环

### Phase 3：富化、聚类、向量化

目标：

- 建立 `feed_enrichments`、聚类、事件簇和 embedding。

状态：

- 未开始正式实施。

### Phase 4：趋势分析与深度总结

目标：

- 将信息流转化为趋势判断、热点迁移和研究型总结。

状态：

- 未开始正式实施。

### Phase 5：知识大脑结构化升级

目标：

- 建立面向世界知识体系的学科结构、权威理解层和前沿研究层。

状态：

- 未开始正式实施。

### Phase 6：进化大脑强化

目标：

- 从日志和接口探针升级到真实 UI 巡检、自动发现、问题归并、恢复与优化建议。

状态：

- 已有基础，但距离目标还有明显差距。

当前已完成：

- 日志监控
- 自测
- 采集健康检查
- 问题中心接入
- MVP 运行状态接口

当前未完成：

- 浏览器级 UI 巡检
- 基于真实体验的自动发现
- 更可靠的优化路径推荐

### Phase 7：分布式部署准备

目标：

- 为未来多机、对象存储、独立 worker 和长期运行做准备。

状态：

- 设计已完成，实施未开始。

## 当前执行顺序

1. 固化本机运行流程和前端质量基线。
2. 将进化大脑做成正式页面和真实质量入口。
3. 回到 Phase 2，继续完成事实层收口。
4. 再推进 Phase 3 和知识大脑相关实施。

## 当前提醒

新的知识存储结构并没有全部完成。已经完成的是原始层和部分事实层；还没有完成的是富化层、聚合层和知识大脑结构化层。
